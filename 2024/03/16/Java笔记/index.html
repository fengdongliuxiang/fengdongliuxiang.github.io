Java 笔记

一、数组

动态初始化：手动指定数组长度，由系统给出默认初始化值

静态初始化：手动指定数组元素，由系统计算数组长度

静态初始化：

int[] array=new int[]{11,22,33};

int[] array={11,22,33};

动态初始化：

int[] array =new int[50];//赋值50个

整数类型默认初始化值为0

小数类型默认初始化值为0.0

字符类型默认初始化值为‘/u0000’空格

布尔类型默认初始化值为false

引用数据类型默认初始化值为null

二、构造方法

1、如果没有定义构造方法，系统将给出一个默认的无参构造方法

2、如果定义了构造方法，不论有参无参，系统都不会再提供默认构造方法

3、带参构造和无参构造可以同名同时存在，即构造方法的重载

4、无论是否使用，都推荐同时定义有参构造和无参构造

三、字符串

StringBuilder

使用场景：

1、字符串拼接

2、字符串反转

字符串原理：

1、直接赋值会复用字符串常量池中的

2、new出来不会复用，而是开辟一个新的空间

3、==号基本数据类型比较数据值，引用类型比较地址值

4、字符串拼接时，如果没有变量参与，都是字符串直接相加，编译之后就是拼接之后的结果，会复用串池中的字符串。

​ 如果有变量参与，每一行拼接的代码，都会在内存中创建新的字符串，浪费内存。

5、StringBuilder提高效率原理：所有要拼接的内容都会往StringBuilder中放，不会创建很多无用的空间，节约内存。

6、StringBuilder源码：

​ 默认创建一个长度为16的字节数组，添加的内容小于16直接存入，大于16会扩容至16*2+2=34，大于34以实际长度为准

四、继承

1、子类不能继承父类的构造方法

2、子类能继承父类的public和private成员变量，但是不能用private成员变量

3、子类能继承父类public成员方法，不能继承private、static、final成员方法（虚方法）

五、多态

1、多态是对象的多种形态

2、多态的前提：

​ ①有继承/实现关系

​ ②有父类引用指向子类对象

​ ③有方法的重写

3、调用成员变量时编译和运行都看左边，即父类如果没有此变量则报错；调用成员方法时编译看左边，运行看右边，即编译代码的时候如果父类没有此方法则报 错，运行代码的时候运行的是子类中的方法

4、多态的优势：方法中，使用父类型作为参数，可以接收所有子类对象

5、多态的弊端：不能使用子类的特有功能

6、引用数据类型的类型转换有自动类型转换和强制类型转换

int i = 123;
long l = i; // 自动将int转换为long

double d = 123.45;
int j = (int)d; // 强制将double转换为int，可能会丢失小数部分

byte b = 10;
int k = b; // 自动将byte转换为int

char c = 'A';
int m = c; // 自动将char转换为int

强制类型转换：

int num1 = 5; //定义一个整数变量num1并初始化为5
double num2 = (double) num1; //使用强制类型转换将整数num1转换成双精度浮点数num2
System.out.println(num2); //输出结果为5.0

六、static关键字

java 静态变量

在Java中，static关键字用于创建类级别的变量和方法，即它们不属于类的任何特定实例，而是属于类本身。

  1. 静态变量（或类变量）

静态变量在内存中只存在一个拷贝，不论这个类实例化了多少对象，它的静态变量只占用一份内存。

静态方法不能使用类的非静态变量，因为非静态变量需要类的特定实例。

七、抽象

在Java中，一个没有具体实现的方法的类可以被定义为抽象类。这个类不能被实例化，只能被继承。继承一个抽象类的子类必须实现父类中的抽象方法。

八、适配器设计模式

当一个接口中抽象方法过多，但是只要使用其中一部分时，可以定义一个中间类XXXAdapter，并重写需要用的方法，为了避免其他类创建适配器类的对象，中间的适配器类用abstract进行修饰。
